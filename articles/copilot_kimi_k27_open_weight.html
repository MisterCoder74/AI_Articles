GitHub Copilot Apre all'Open-Weight: Kimi K2.7 Code nel Model Picker

GitHub Copilot integra Kimi K2.7 Code: primo open-weight nel picker, architettura MoE da 1T parametri, governance opt-in e fine di GitHub Models.

Vivacity AI Division  ·  Luglio 2026  ·  Alessandro Demontis

I. 1° Luglio 2026: Copilot Apre la Porta all'Open-Weight

Il 1° luglio 2026 non è una data qualsiasi nel calendario dell'AI applicata allo sviluppo software. GitHub ha reso Kimi K2.7 Code — il modello di Moonshot AI — generalmente disponibile nel model picker di GitHub Copilot. Per la prima volta nella storia dello strumento, un modello open-weight è selezionabile dall'utente all'interno della piattaforma di coding AI più diffusa al mondo.

La mossa non è cosmética. Segnala una trasformazione strutturale: GitHub non è più un distributore di modelli proprietari di frontiera. Diventa un orchestratore multi-laboratorio, capace di instradare le richieste degli sviluppatori verso cinque provider distinti sotto un'unica sottoscrizione. Per i 4,7 milioni di abbonati paganti censiti a gennaio 2026, il model picker è cambiato aspetto — e sostanza — in un solo mercoledì mattina.

Per la prima volta, un modello open-weight entra nell'assistente di coding più diffuso al mondo: non come esperimento, ma come opzione di produzione selezionabile in produzione da milioni di sviluppatori.

II. Diciannove Giorni: da Hugging Face al Model Picker di Copilot

La velocità è la prima cosa che colpisce. Moonshot AI ha pubblicato i pesi di Kimi K2.7 Code su Hugging Face in giugno 2026. Diciannove giorni dopo, il modello era già selezionabile dentro GitHub Copilot: uno dei cicli di integrazione più rapidi mai registrati tra rilascio open-weight e disponibilità su piattaforma enterprise.

Il percorso è stato tutt'altro che automatico. GitHub ha dovuto validare il modello, costruire l'integrazione con il layer AI Credit introdotto il 1° giugno 2026, definire le superfici supportate — da Visual Studio Code 1.127.0+ a JetBrains 1.9.1-251+, passando per GitHub Mobile su iOS e Android — e, soprattutto, progettare la policy di governance per i piani aziendali. Diciannove giorni per tutto questo: un segnale che la pipeline di onboarding dei modelli in Microsoft Azure si sta consolidando in qualcosa di industrialmente scalabile.

19 giorni: l'intervallo tra la pubblicazione dei pesi di Kimi K2.7 Code su Hugging Face e la sua disponibilità nel model picker di GitHub Copilot — una delle transizioni open-weight → enterprise più rapide mai documentate.

III. Un Trilione di Parametri, 32 Miliardi Attivi: Come Funziona il MoE di K2.7 Code

Kimi K2.7 Code è costruito su un'architettura Mixture-of-Experts. I numeri di superficie sembrano contraddittori: un trilione di parametri totali, un costo di inferenza paragonabile a modelli proprietari molto più piccoli. La contraddizione si risolve guardando dentro il meccanismo.

Il modello è composto da 384 esperti specializzati, distribuiti su 61 layer transformer. Per ogni token processato, un router appreso seleziona soltanto 8 esperti attivi più 1 esperto condiviso. Il risultato: ogni passaggio di inferenza calcola attraverso 32 miliardi di parametri attivi — il comportamento computazionale di un modello denso da 32B — mentre la rete conserva la capacità rappresentativa dell'intero pool da un trilione di parametri.

1 trilione di parametri totali, 32 miliardi di parametri attivi per token: l'architettura Mixture-of-Experts di Kimi K2.7 Code con 384 esperti rende economicamente sostenibile un modello di capacità frontier a un prezzo inferiore ai modelli proprietari concorrenti.

Il punto editoriale che vale la pena fissare: il basso costo di Kimi K2.7 Code nel sistema AI Credit di GitHub non è il risultato di un sussidio commerciale. È la conseguenza diretta di una scelta ingegneristica. La distinzione tra "costa poco perché qualcuno ci rimette" e "costa poco perché l'architettura lo rende efficiente" è centrale per valutare la sostenibilità del modello nel lungo periodo — specialmente per chi deve giustificare la scelta a un CFO o a un comitato di approvvigionamento.

Quando l'efficienza è un'architettura e non un sussidio, il prezzo basso diventa strutturale: per i tech lead che devono difendere la scelta in un budget review, questa distinzione vale più di qualsiasi benchmark.

IV. La Clausola che gli Amministratori Non Possono Ignorare

GitHub ha impostato Kimi K2.7 Code come disabilitato per default nei piani Copilot Business ed Enterprise. Un amministratore del piano deve attivare esplicitamente la policy nelle impostazioni prima che chiunque nell'organizzazione possa selezionare il modello. È la prima volta che questa logica opt-in viene applicata a un modello nel model picker di Copilot.

La ragione è precisa. Moonshot AI è un'azienda cinese soggetta alla National Intelligence Law della RPC. I prompt degli utenti vengono instradati verso l'infrastruttura di Microsoft Azure negli Stati Uniti — non verso i server di Moonshot AI — e questo cambia dove i dati transitano al momento dell'inferenza. Non cambia, però, cosa Pechino può legalmente richiedere a Moonshot AI come entità giuridica. La distinzione è sottile ma rilevante per qualsiasi valutazione seria di compliance.

Per gli amministratori enterprise, le domande da porre prima di abilitare la policy sono concrete: i dati dei prompt rientrano nel perimetro normativo del proprio settore? Esiste una policy aziendale sull'utilizzo di infrastrutture con fornitori soggetti a giurisdizioni extra-UE ed extra-USA? I contratti con i clienti finali contengono clausole sul trattamento dei dati che potrebbero essere impattate? La risposta tecnica di GitHub — routing su Microsoft Azure — è necessaria ma non sufficiente per rispondere a tutte e tre.

V. Cinque Provider, un Solo Abbonamento: il Model Picker Cambia Forma

Con l'ingresso di Moonshot AI, il model picker di GitHub Copilot copre ora cinque laboratori AI indipendenti: OpenAI, Anthropic, Google, Microsoft e Moonshot AI. Nessun altro strumento di coding sul mercato instrada attualmente su cinque provider distinti sotto un'unica sottoscrizione.

Il sistema AI Credit — introdotto il 1° giugno 2026 a $0,01 per credito — rende il costo comparabile in modo trasparente. Kimi K2.7 Code è fatturato a "provider list pricing", nella fascia bassa rispetto ai modelli di frontiera già presenti. Per un team che usa Copilot su scala, la scelta del modello non è più solo una decisione tecnica: diventa una leva di ottimizzazione dei costi con una granularità che prima non esisteva.

L'implicazione strategica per i tech lead è altrettanto rilevante sul piano del lock-in: avere cinque provider sotto un abbonamento non elimina la dipendenza dalla piattaforma GitHub, ma riduce sensibilmente la dipendenza da un singolo laboratorio AI. Un cambiamento di pricing di OpenAI o un'interruzione del servizio di Anthropic non blocca più il workflow: c'è un fallback selezionabile in pochi clic.

VI. Il Contrappeso: GitHub Models Chiude il 30 Luglio

Lo stesso 1° luglio 2026 in cui Kimi K2.7 Code entrava nel picker, GitHub annunciava il ritiro definitivo di GitHub Models per il 30 luglio 2026. La timeline è già in corso: dal 16 giugno 2026, la piattaforma non accetta più nuovi clienti — né free né paid. Il 30 luglio chiuderanno playground, catalogo modelli, API di inferenza e BYOK per tutti gli utenti esistenti.

Per i team che usavano GitHub Models per prototipazione rapida, la finestra di migrazione è stretta. I percorsi indicati da GitHub sono due: Copilot per i workflow di coding agentico, Azure AI Foundry per l'inferenza programmatica generale. Attenzione particolare va riservata alle pipeline CI/CD, agli hook di automazione e ai flussi di credenziali che dipendono dalle API di GitHub Models: vanno aggiornati prima della scadenza, non dopo.

Letta insieme all'apertura di Copilot ai modelli open-weight, la chiusura di GitHub Models rivela la direzione strategica di GitHub con chiarezza: consolidare il traffico di inferenza su Copilot e su Azure AI Foundry, eliminando una superficie separata che frammentava l'esperienza e i dati di utilizzo. Il risultato per gli sviluppatori è un ecosistema più compatto — e per Microsoft Azure, una concentrazione maggiore dei flussi di inferenza sotto il proprio ombrello commerciale.
